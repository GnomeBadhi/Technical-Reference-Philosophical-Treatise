# Contributing to The Sovereignty Engine

Thank you for your interest in this project. Because this work is released under the **CC BY-NC-ND 4.0** license, the types of contributions that are accepted are intentionally narrow. Please read this guide before opening an issue or reaching out.

---

## What You Can Contribute

| Contribution type | Accepted? |
|-------------------|-----------|
| Bug reports (broken links, typos, rendering issues) | ✅ Yes |
| Clarification questions about the documentation | ✅ Yes — open an Issue |
| Suggestions for additional index entries or navigation improvements | ✅ Yes |
| Translation proposals | ⚠️ Contact the author first |
| Code forks or derivative works | ❌ No — requires explicit author approval |
| Commercial use proposals | ❌ No — prohibited by license |

---

## Opening an Issue

1. Check the [existing issues](https://github.com/GnomeBadhi/Technical-Reference-Philosophical-Treatise/issues) to avoid duplicates.
2. Use a clear title that summarises the problem (e.g. `Broken link in docs/INDEX.md → diagrams/Regime-Map.jpg`).
3. Include:
   - The file and section where you found the problem.
   - What you expected to see.
   - What you actually saw (paste the relevant text or attach a screenshot).

---

## Proposing Adaptations or Derivatives

The "Ethical Gatekeeper" policy means that any fork, translation, or modified version of the Sovereignty Engine requires the author's written approval.

To make such a proposal:
1. Open an Issue titled `[Adaptation Proposal] <brief description>`.
2. Describe the intended use, audience, and how it relates to the original work.
3. Confirm that the proposed use is non-commercial and that attribution will be preserved.

The author will respond to all adaptation proposals.

---

## Style Guide (for documentation PRs)

If a documentation fix is approved and you are invited to submit a pull request:

- Write in plain, concise English.
- Keep file names consistent with existing conventions (Title-Case for diagrams, UPPER_CASE for spec directories, `README.md` for per-directory readmes).
- Update `docs/INDEX.md` if you add or move any file referenced there.
- Do not add new dependencies to the DEMO without prior discussion.
