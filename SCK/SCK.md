## 1. Concept Overview (Minimal Disclosure)

The **Sovereign Commerce Kernel (SCK)** is a formal architecture defining value transformation using a triad of primitives. These primitives form the irreducible basis for computing influence on value within a sovereign, auditable system.

### **ΔT — Tangible Work Unit**  
A conserved physical or measurable change.  
Constraint:  
\[
\sum ΔT_{\text{system}} = 0
\]

### **ΔA — Attention Influence Unit**  
A non‑conserved human attention or participation signal.

### **Γ — Agreement Unit**  
A non‑conserved human assent signal representing legitimacy of value change.

These three primitives constitute the complete conceptual basis for the SCK.

## 2. Core Invariants (Public Minimal Set)

The SCK is governed by four invariants:

1. **No value without tangible linkage**  
2. **No value change without attention**  
3. **No value change without agreement**  
4. **All events must be auditable and non‑interruptible**

These invariants define the boundary conditions for any valid SCK implementation.

## 3. Influence Function (Minimal Public Form)

The kernel computes influence using the following structure:

\[
I = αΔT + βh(ΔA) + γΓ
\]

Where:

- \(α, β, γ\) are normalization constants  
- \(h\) is a monotonic mapping from attention to influence  

This expression defines the minimal public form of the influence mechanism.

---

## 4. Kernel Value Delta (Minimal Public Form)

Aggregate value change is defined as:

\[
ΔK = \sum I
\]

This establishes the kernel’s value state without revealing internal mechanisms, node topology, or ledger integration details.
